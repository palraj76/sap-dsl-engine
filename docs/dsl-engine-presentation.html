JSON DSL Payload Structure

{ "version": "1.3", "query_id": "Q-001", // Raw mode "sources": [{"table":"USR02", "alias":"u"}], "joins": [{"type":"left", ...}], // OR Entity mode "entity": "user_access", "select": [{"field":"u.BNAME", ...}], "filters": {"logic":"AND", ...}, "group_by": ["u.BNAME"], "metrics": [{"type":"count", ...}], "having": [{"metric":"cnt", ...}], "order_by": [{"field":..., "direction":...}], "limit": {"rows":100, "page_size":50}, "params": {"asOfDate": "20260324"}, "output": {"include_summary": true} }

Key Features

  • Two modes: Raw (sources+joins) or Entity (logical name)
  • Condition trees: AND/OR nesting at any depth
  • Type annotations: STRING, DATE, AMOUNT, QUANTITY
  • Param binding: Safe value injection
  • Pagination: page_size + page_token
  • Aggregations: COUNT, SUM, AVG, MIN, MAX

Supported Operators

= != > < >= <= IN NOT IN IS NULL IS NOT NULL BETWEEN

Security Architecture

Layer 1: Network

  • HTTPS only (TLS 1.2+)
  • VPN or IP whitelist
  • Pull-only (SAP never calls out)
  • Client firewall controlled

Layer 2: Authentication

  • SHA-256 hashed credentials
  • Short-lived bearer tokens (1h)
  • X-DSL-Token custom header
  • Per-client credential isolation

Layer 3: Authorization

  • WHITELIST_MODE: STRICT / OPEN
  • Field-level whitelist (ZJSON_DSL_WL)
  • Wildcard (*) support per table
  • Role-based access (ZDSL_AUDIT)

Layer 4: Validation

  • Field pattern: alias.FIELD regex
  • Operator allow-list only
  • Value length cap (500 chars)
  • SQL injection detection

Layer 5: Guardrails

  • MAX_ROWS_ALLOWED: 10,000
  • MAX_TIMEOUT_SEC: 30
  • Join count warnings
  • Large table pagination alerts

Layer 6: Audit

  • Every query logged (ZJSON_DSL_AUDIT)
  • Every field access logged (ZJSON_DSL_ALOG)
  • SELECT / FILTER / JOIN tracking
  • Client-facing transparency reports

SQL Syntax Modes

SQL Generation — Dual Syntax

Single Table (Old Syntax)

-- No JOINs: space-separated, INTO before FROM SELECT u~BNAME u~USTYP INTO TABLE <lt_result> UP TO 10 ROWS FROM USR02 AS u WHERE u~USTYP = 'A'

Compatible with all ABAP releases (ECC 6.0+)

JOINs (New Syntax)

-- JOINs: comma-separated, @ escape, no MANDT in ON SELECT u~BNAME, ru~AGR_NAME, ru~TO_DAT FROM USR02 AS u LEFT OUTER JOIN AGR_USERS AS ru ON u~BNAME = ru~UNAME WHERE u~USTYP = 'A' INTO TABLE @<lt_result> UP TO @lv_limit ROWS

Required by ABAP 740+ for dynamic JOIN clauses

MANDT automatically handled by SAP in new syntax — stripped from JOIN ON by the builder
